## Сайт «ИТА Сочи»

Статический сайт системного интегратора «ИТА Сочи» с услугами по монтажу СКС, ВОЛС, видеонаблюдения, СКУД, IT‑аутсорсингу и т.д.

Сайт собран по техническому описанию из файла `web ita-sochi.txt`.

### Технологии

- Чистый **HTML/CSS/JS**
- **Tailwind CSS** и **Font Awesome** через CDN
- Один PHP‑скрипт `submit.php` для обработки форм (отправка писем и логирование)

### Структура проекта

- `index.html` — главная страница
- `pages/uslugi.html` — услуги
- `pages/projects.html` — проекты
- `pages/catalog.html` — каталог оборудования
- `pages/about.html` — о компании
- `pages/blog.html` — блог и кейсы
- `pages/contacts.html` — контакты
- `privacy-policy.html` — политика конфиденциальности
- `css/style.css` — стили
- `js/main.js` — скрипты (анимации, валидация формы, AJAX‑отправка)
- `submit.php` — обработчик форм (email + лог в `logs/requests.log`)
- `.htaccess`, `robots.txt`, `sitemap.xml` — сервисные файлы

### Запуск локально

Простой просмотр (без работы PHP‑форм):

1. Откройте `index.html` в браузере двойным кликом или через «Open with Live Server» в VS Code/Cursor.

Для теста отправки форм (`submit.php`) нужен локальный веб‑сервер с PHP, например:

```bash
php -S localhost:8000
```

И зайти на `http://localhost:8000/index.html`.

### Деплой на GitHub Pages

GitHub Pages — **только статический хостинг**, PHP на нём не выполняется.  
Это значит:

- Страницы HTML/CSS/JS будут работать.
- `submit.php` на GitHub Pages выполняться не будет (формы отправляться не смогут).

Чтобы выложить именно статическую часть:

1. В репозитории [`web-ita-sochi`](https://github.com/akoffice933-maker/web-ita-sochi) зайдите в **Settings → Pages**.
2. В блоке **Build and deployment**:
   - **Source**: выберите **Deploy from a branch**.
   - **Branch**: `main`, папка `/ (root)`.
3. Нажмите **Save**.

Через 1–2 минуты сайт будет доступен по адресу:

```text
https://akoffice933-maker.github.io/web-ita-sochi/
```

> Для работоспособности форм на GitHub Pages можно:
> - либо использовать внешний сервис форм (Formspree, Netlify Forms и др.),
> - либо разместить `submit.php` на отдельном хостинге и прописать в `action` формы URL на этот сервер.

### Деплой на обычный хостинг (c PHP)

Чтобы формы работали через `submit.php`, нужен хостинг с поддержкой PHP (любой shared‑хостинг, VPS, VDS и т.п.).

Общий порядок:

1. Создайте домен/поддомен на хостинге (например, `ita-sochi.ru` или `test.ita-sochi.ru`).
2. Загрузите **все файлы из репозитория** в корневую папку сайта (обычно `public_html`, `www`, `htdocs` и т.п.).
3. Убедитесь, что:
   - `index.html` лежит в корне сайта,
   - `submit.php` тоже в корне,
   - структура папок `css`, `js`, `pages`, `logs` сохранена.
4. При необходимости в `submit.php` поменяйте адрес получателя:

```php
$to = "info@ita-sochi.ru";
```

5. Откройте сайт в браузере по домену — формы должны отправляться, а заявки логироваться в `logs/requests.log`.

### Дальнейшая работа

- Редактирование прямо в GitHub → потом `git pull` локально.
- Локальные правки → `git add`, `git commit`, `git push`.

Если понадобится:
- добавить англоязычную версию,
- докрутить SEO/микроразметку,
- адаптировать под другой домен/бренд —
можно расширять текущую структуру, не ломая существующие страницы.
