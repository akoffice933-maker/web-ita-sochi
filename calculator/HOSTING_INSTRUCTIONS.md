### Если на хостинге есть Composer:

1. **Подключитесь по SSH:**
   ```bash
   ssh infoitaso3@school-board.online
   ```

## 🔧 Шаг 8: Настройка .htaccess (если нужно)

### Если не работает API:

1. **Откройте файл** `calculator/.htaccess`

2. **Убедитесь, что там есть:**

```apache
RewriteEngine On
RewriteBase /calculator/

# Перенаправление API запросов
RewriteRule ^api/(.*)$ api/index.php/$1 [QSA,L]

# Для SPA - все запросы перенаправлять на index.html
RewriteCond %{REQUEST_FILENAME} !-f
RewriteCond %{REQUEST_FILENAME} !-d
RewriteRule ^(.*)$ index.html [QSA,L]
```

3. **Измените `RewriteBase`** на ваш путь:
   - Если в корне: `RewriteBase /`
   - Если в папке: `RewriteBase /calculator/`

## 🔐 Шаг 9: Безопасность

### Обязательно сделайте:

1. **Удалите config.php из репозитория GitHub:**
   ```bash
   git rm --cached calculator/config.php
   echo "calculator/config.php" >> .gitignore
   git commit -m "Remove config.php from repo"
   git push
   ```

2. **Установите права на файлы:**
   - Файлы: `644`
   - Папки: `755`
   - config.php: `600` (только чтение для владельца)

3. **Смените пароли:**
   - От FTP
   - От базы данных
   - От панели хостинга

4. **Закройте доступ к служебным файлам:**

Добавьте в `.htaccess`:

```apache
# Запрет доступа к конфигам
<FilesMatch "(config|example|sql)$">
    Order allow,deny
    Deny from all
</FilesMatch>
```

## 🆘 Шаг 10: Решение проблем

### Страница не загружается (404):

- Проверьте путь к папке
- Проверьте `.htaccess`
- Попробуйте `https://school-board.online/calculator/index.html`

### Ошибка подключения к БД:

- Проверьте логин/пароль в `config.php`
- Убедитесь, что база создана
- Проверьте имя пользователя БД

### Не работает API:

- Проверьте `.htaccess`
- Включите `APP_DEBUG = true` в `config.php`
- Проверьте логи ошибок PHP

### Не работает экспорт PDF/Excel:

- Установлен ли Composer?
- Загружена ли папка `vendor/`?
- Выполните `composer install` на хостинге
